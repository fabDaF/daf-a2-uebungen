# A2.1 — Masterplan für interaktive HTML-Übungsdateien

Dieses Dokument ist die verbindliche Referenz für alle A2.1-Dateien.
Es hält fest, was bereits erarbeitet wurde, welche Dateitypen es gibt,
und wie jede neue Datei aufgebaut werden muss.

---

## 1. Quelle und Nomenklatur

**Quelle:** Lingoda-PDFs unter `Cowork/lingoda/A 2.1/`

**Dateiname-Schema der Lingoda-PDFs:**
```
A2_LLLLX_DE.pdf
     ^^^^
     ||||__ X = Datei-Typ (V, X, G, R, S, W)
     |||___ Lektionsnummer innerhalb der Einheit (1–8)
     ||____ Einheitsnummer (01–07)
     |_____ Kurs (1 = A2.1)
```

**Dateiname-Schema der HTML-Ausgabedateien:**
```
DE_A2_LLLLx-thema.html
```
Beispiel: `DE_A2_1011V-meine-freunde.html`

## 3. Datei-Typen und ihre Tab-Struktur

Jede Lingoda-Lektion hat einen Buchstaben-Code. Dieser bestimmt den Tab-Aufbau der HTML-Datei.

### V — Vokabular (`*V*.html`)

**Zweck:** Neue Wörter mit Fotos, Kollokationen und Übungen einführen.

**Tab-Struktur:**
| Tab | Emoji | Name | Inhalt |
|-----|-------|------|--------|
| 0 | 📸 | Wörter | Foto-Vokabelkarten (10–12 Karten): Artikel · Wort · Plural · englische Übersetzung · **Beispielsatz aus Quasthoff-Kollokationen** |
| 1 | 💬 | Redemittel | Frage-und-Antwort Drag & Drop (Skill: daf-uebungsformen) |
| 2 | ✏️ | Lückentext | 2 Lückentexte in einem Tab, Live-Feedback, Timer |
| 3 | 📖 | Text & Fragen | Lesetext + Richtig/Falsch-Aufgaben |
| 4 | 🧩 | Satzbau | 6–8 Sätze, Drag & Drop (Skill: satzbau-drag-drop), Timer |
| 5 | 🔠 | Wortschatz | Alle Vokabeln: Artikel + Wort + Plural tippen, Live-Feedback, Timer |

**Referenzdatei (genehmigt):** `DE_A2_1011V-meine-freunde.html`

---

### X — Extra-Übungen (`*X*.html`)

**Zweck:** Vertiefung und Anwendung des Vokabulars aus der V-Datei.

**Tab-Struktur:** *(Platzhalter — wird beim ersten X-File definiert)*

Erwartete Übungstypen: Zuordnung, Lückentext mit höherem Schwierigkeitsgrad,
Schreibaufgaben, situative Dialoge.

### G — Grammatik (`*G*.html`)

**Zweck:** Grammatikregel entdecken, verstehen und üben.

**Tab-Struktur:** *(Skill: daf-grammatik-uebungen — vor Erstellung lesen)*

Typische Tabs:
| Tab | Emoji | Name | Inhalt |
|-----|-------|------|--------|
| 0 | 🔍 | Entdecken | Beispielsätze, Regel ableiten |
| 1 | 📋 | Regel | Regelkarte, Paradigmentabelle |
| 2 | ✏️ | Lückentext | Grammatik-Lückentext, Live-Feedback, Timer |
| 3 | 🧩 | Satzbau | Sätze mit Grammatikfokus ordnen, Timer |
| 4 | 🎯 | Multiple Choice | Richtig/Falsch oder A/B/C-Auswahl |

**Zweck:** Lesetext mit Vorentlastung, Verstehensaufgaben und Wortschatzübungen.

**Tab-Struktur:** *(Skill: a1-story-uebungen — vor Erstellung lesen und adaptieren)*

|-----|-------|------|--------|

| 2 | ❓ | Verständnis | Richtig/Falsch + offene Fragen |
| 3 | ✏️ | Lückentext | Sätze aus dem Text mit Lücken |
| 4 | 🧩 | Satzbau | Sätze aus dem Text zerlegt, Timer |
| 5 | 🔠 | Wortschatz | Schlüsselwörter tippen (Artikel + Wort + Plural), Timer |

## 4. Verbindliche Qualitätsregeln (für ALLE Dateitypen)

### Layout
- Lila Hintergrund-Gradient (`#667eea` → `#764ba2`) — Skill: `daf-html-layout`
- Weißer Container, max. 1000px, Schatten
- Nav-Buttons: Emoji **über** Text (`nav-emoji` + `nav-label`, `flex-direction: column`)
- Aktiver Tab: Klasse `active` (nicht `aktiv`); sichtbare Section: Klasse `aktiv`

### Lücken-Inputs — Placeholder IMMER leer

⛔ **`placeholder="___"` ist verboten.** Niemals Striche, Punkte oder andere Füllzeichen.

```html
<!-- ❌ FALSCH -->
<input placeholder="___">
<!-- ✅ RICHTIG -->
<input placeholder="">
```

Diese Regel wurde mehrfach explizit festgelegt und darf nie wieder gebrochen werden.

### Live-Feedback bei Texteingaben
Gilt für **alle** Tipp-Felder — Skill: `daf-uebungsformen` (Abschnitt „Live-Feedback")

| Zustand | Farbe |
|---------|-------|
| Leer | grau (neutral) |
| Prefix korrekt | lila (`:focus`) |
| Vollständig korrekt | grün (`.ok`) |
| Falsch | rot (`.no`) |

**Kein Prüfen-Button. Niemals.**

### Timer
- Tabs mit Tipp- oder Drag-Übungen bekommen einen Timer
- Timer-Zeile: `⏱ Zeit · 🏆 Bestzeit · ↺ Neustart · 💡 Lösungen` — alles in **einer Zeile** (`.btn-row`)
- Timer startet bei erster Interaktion (`timerAutoStart`), stoppt wenn alle korrekt (`timerStop`)
- Bestzeit wird in `localStorage` gespeichert (Key: `TIMER_PREFIX-best-N`)

### Satzbau
Verbindlich: Skill `satzbau-drag-drop` — `sbDragged`-Variable, `sb-bank-N`/`sb-row-N`/`sb-fb-N`-IDs, Fisher-Yates-Shuffle, 500ms-Timeout vor Farb-Feedback

### Bilder (Pexels)
URL-Schema:
```
https://images.pexels.com/photos/ID/pexels-photo-ID.jpeg?auto=compress&cs=tinysrgb&w=400&h=300&fit=crop
```
Bilder-IDs per WebSearch suchen (`site:pexels.com/photo BESCHREIBUNG`).
Bilder müssen zum Wort passen — nie blind verwenden.

### Beispielsätze (nur V-Dateien)
Jeder Beispielsatz auf einer Vokabelkarte muss eine **belegte Kollokation** aus dem Quasthoff-Wörterbuch zeigen:
`Cowork/Grammatik/Worterbuch-Der-Kollokationen.txt`

Verben und Adjektive aus `V:AKK`, `V:SUBJ`, `A:`-Einträgen wählen.
Keine Allerweltsformulierungen wie „X ist schön" oder „Y ist wichtig".

### Copyright
Footer immer exakt so:
```html
<span class="author-name">&copy; Frank Burkert</span> &middot; DaF-Materialien &middot;
<a href="mailto:FrankBurkert@fabdaf.onmicrosoft.com">FrankBurkert@fabdaf.onmicrosoft.com</a>
```

⛔ **ABSOLUTES TABU — niemals ändern, niemals „verbessern", niemals durch eine andere Variante ersetzen.**
Der Footer ist das Aushängeschild des Autors. Keine Skill-Regel, kein Qualitätscheck und keine eigene Initiative rechtfertigt eine Änderung. Nur eine explizite schriftliche Anweisung des Benutzers darf den Footer-Text verändern.

## 5. Arbeitsreihenfolge beim Erstellen einer neuen Datei

1. **Lingoda-PDF lesen** (`pdftotext` → vollständigen Inhalt extrahieren)
2. **Passende Skills lesen** (je nach Datei-Typ)
3. **Bilder suchen** (WebSearch mit `site:pexels.com/photo`)
4. **Kollokationen prüfen** (Quasthoff, nur für V-Dateien)
5. **Datei aufbauen** — Tab für Tab, Skill-Muster einhalten
6. **Selbst-Check vor Auslieferung:**
   - [ ] Emojis über Text in Nav-Buttons?
   - [ ] Kein Prüfen-Button?
   - [ ] Live-Feedback korrekt implementiert (erst `val === ans`, dann `startsWith`)?
   - [ ] Timer-Zeile vollständig (Neustart + Lösungen in einer Linie)?
   - [ ] Satzbau nach Skill?
   - [ ] Copyright unverändert?
   - [ ] Beispielsätze aus Quasthoff belegt?
